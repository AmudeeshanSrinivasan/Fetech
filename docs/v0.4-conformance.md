# v0.4 document, media, cache, snapshot, and archive development conformance

The unreleased v0.4 development overlay covers the final three catalogue categories at the
runtime-contract level: 36/36 v0.4 entries have an importable native path or a typed optional
dependency/provider boundary. Together with v0.1-v0.3 this gives 155/155 registered implementation
paths across all 13 categories.

`closure_ready=true` means there are no `planned` entries in the conformance overlay. It does not
mean every optional binary or operator connector is installed, and it is not by itself a production
or release-readiness claim. The package version remains `0.3.0a0`; no v0.4 tag or release artifact
exists yet.

Capability documents expose `implementation_available` separately from `runtime_available`.
The former closes the catalogue path; the latter is true only for capabilities guaranteed by the
required runtime. Optional, target-dependent, binary-backed, and configured-provider paths remain
runtime-unavailable until an interface-specific availability probe is added.

## Capability accounting

| Category | Native | Optional | Total | Runtime owner |
| --- | ---: | ---: | ---: | --- |
| Documents and files | 8 | 6 | 14 | `DocumentAdapter` and `ArchiveAdapter` |
| Images, audio, video, and media | 2 | 9 | 11 | `MediaAdapter` |

The 19 optional paths fail as `DEPENDENCY_MISSING` when their reviewed dependency or injected
provider is absent. They never silently fall back to an unrelated capability and never mark an
unverified result as accepted.

## Deterministic planning and interfaces

Explicit canonical output IDs are considered before filename heuristics. One request may schedule
multiple independent media extractors or cache strategies. Previous-snapshot and configured archive
alternatives are policy-gated before HTTP so they can recover from an unavailable origin; failed or
low-quality alternatives fall through deterministically. HTTP remains in the plan for mixed
content-and-cache requests, while local and representation-specific cache writes run only after the
requested extractor has produced an accepted artifact of the exact representation.

Document, media, archive, and connector attempts use the request's remaining wire,
decompressed-byte, archive-member, compression-ratio, and deadline budgets rather than treating each
node's reservation as a fresh allowance. Attempts report actual consumption so the executor can
decrement budgets cumulatively and stop with `BUDGET_EXHAUSTED`.

One gateway-owned scheduler admits primary HTTP, built-in Wayback, and built-in yt-dlp operations.
HTTP and Wayback keep DNS/policy evaluation and the corresponding request in the same slot, enforce
one cumulative direct-adapter deadline, record partial/failure bytes, bound host-pacing history, and
release capacity synchronously under cancellation. The yt-dlp slot covers the complete subprocess
operation, not each internal request to its several allowed hosts; publication wording and tests
keep that distinction explicit.

The SDK, CLI, REST daemon, and MCP tools all use the same `UniversalFetchGateway`. The gateway owns
these execution adapters:

```text
policy
  ├─ previous snapshot / configured archive alternative ──> CacheAdapter
  └─ HTTP raw artifact
       ├─ document ID ──> DocumentAdapter ──> bounded DocumentParseWorker
       ├─ zip_archive ──> ArchiveAdapter ───> bounded ArchiveParseWorker
       ├─ media ID ─────> MediaAdapter ─────> native parser / bounded tool / provider
       └─ accepted representation ──────────> SnapshotStore
```

The optional Clingo planner can only accept the same registered node structure after Python schema,
availability, policy, dependency, and budget validation. Prolog explanations do not execute
document, media, or cache operations.

## Document worker boundary

`DocumentAdapter` sends acquired bytes, a path suffix, and numeric limits to
`python -m fetech.document_worker`. Its request protocol omits hosts, query strings, credentials,
opaque auth references, and caller target paths. When the preferred Docling path is configured, it
also carries one parent-validated absolute path to an explicitly provisioned local model-artifact
directory and one independently configured expected bundle SHA-256. The parent and child require a
canonical manifest, hash every bounded file, and reject a digest mismatch before Docling admission;
the model directory is then allowlisted read-only by the Python audit policy. Before Docling imports,
the child installs an audit hook that denies Python-level network/process operations, filesystem
mutation, and reads outside reviewed interpreter/package/model roots. The Pillow image worker
applies the same policy after loading reviewed decoder plugins. These hooks are bypassable by native
code. In development mode, each child is still an ordinary operating-system process without
syscall, namespace, general filesystem, or kernel network containment; Linux required mode supplies
the separate covered-worker boundary described below. The parent process independently recomputes
the signature-based format and validates:

- the canonical capability and parser identity;
- document type, block schema, locator family, and locator equality;
- input, normalized-output, depth, block, member, and compression limits;
- strict JSON without duplicate keys or non-finite numbers;
- XML without DTD or entity declarations;
- OOXML container identity;
- ZIP member names, duplicates, links, encryption, nesting, ratios, and expansion;
- a SHA-256/size-only representation for unknown binary datasets.

PDF, DOCX, PPTX, and XLSX use the optional `fetech[documents]` engines. That extra pins
`docling-slim[convert-core,format-pdf,models-local]==2.113.0`; the `convert-core` feature supplies the
local `DocumentConverter` used by this path. PDF parsing prefers Docling only with an explicit local
artifact directory and expected digest; remote services, external plugins, implicit model
downloads, OCR, enrichments, and image generation are disabled. Non-success, timeout/error, sparse,
or incomplete conversion results are rejected rather than accepted as complete documents. A PDF
with no extractable text returns a checked-only document with `PageState.NEEDS_OCR` when no PDF OCR
provider is configured. With an injected provider, page-located OCR output is type-, locator-,
page-count-, and byte-validated before the document can be accepted.

### Reference Docling artifact workflow

The v0.4 reference bundle contains `docling-project/docling-layout-heron` at immutable Hugging Face
revision `8f39ad3c0b4c58e9c2d2c84a38465abf757272d8`. Its canonical bundle SHA-256 is
`e9aab284777b02541f427ff10ff7e2f1b5656eda04afa3082b9b448d8201bd76`. The manifest records the
published model-card license identifier `apache-2.0`; that record does not replace human notice and
redistribution review.

Provisioning is explicit and networked:

```bash
uv sync --extra documents
uv run python scripts/provision_docling_artifacts.py \
  --output-dir runtime-data/docling-models/2.113.0 \
  --cache-dir runtime-data/docling-download-cache \
  --revision 8f39ad3c0b4c58e9c2d2c84a38465abf757272d8 \
  --expected-sha256 e9aab284777b02541f427ff10ff7e2f1b5656eda04afa3082b9b448d8201bd76
```

The provisioner refuses to overwrite an existing root, verifies the canonical content manifest
against the independent expected digest, removes write bits, and verifies the final tree again.
`runtime-data/` is ignored by Git and neither the model tree nor Hugging Face cache is packaged.
The same script has a network-free verification mode:

```bash
uv run python scripts/provision_docling_artifacts.py \
  --verify-only \
  --output-dir runtime-data/docling-models/2.113.0 \
  --expected-sha256 e9aab284777b02541f427ff10ff7e2f1b5656eda04afa3082b9b448d8201bd76
```

Production deployment should set both values explicitly:

```bash
export FETECH_DOCLING_ARTIFACTS_PATH="$PWD/runtime-data/docling-models/2.113.0"
export FETECH_DOCLING_ARTIFACTS_SHA256="e9aab284777b02541f427ff10ff7e2f1b5656eda04afa3082b9b448d8201bd76"
```

When only the environment path is set, the environment loader supplies the compiled v0.4 reference
digest, so that fallback admits only the exact reference bundle. A custom bundle or direct SDK
construction must provide its own matching expected digest; a digest declared by the model
directory is never trusted.

This local workflow establishes package/model identity and offline conversion behavior only. The
Python audit hook remains bypassable by native code, and a development-mode smoke is not evidence of
Linux namespaces, mounts, seccomp, cgroups, or network denial. Release handling of hostile PDFs must
also pass the Linux `document_parser` required-containment gate with a root-owned read-only copy of
the same digest-bound model tree.

A Git LFS v1 pointer requires a separately configured resolver. The resolver receives a sanitized
target, canonical origin, object SHA-256/size, deadline, and byte ceiling; Python rejects a
different origin, wrong result type, size/hash mismatch, timeout, or provider error. `github_raw`
similarly requires the exact `https://raw.githubusercontent.com` origin and a bounded repository
path.

## Media boundaries

`MediaAdapter` admits only schema-validated output:

- structurally checked PNG, GIF, JPEG, TIFF, and WebP metadata with a configurable pixel ceiling;
- DTD/entity-free podcast RSS with byte, XML-node, depth, and episode limits, plus bounded
  VTT/SRT/text transcripts;
- `TesseractOCRWorker`, `FFprobeWorker`, and `FFmpegThumbnailWorker` through fixed executable
  argument vectors and bounded stdin/stdout;
- an injected `TranscriptProvider` and the optional built-in `YTDLPMetadataWorker`.

EXIF output omits GPS, camera owner, serial, user-comment, and MakerNote fields. The built-in live
YouTube worker receives only an exact supported origin and allowlisted locator query keys;
signatures, tokens, and downloader URLs are rejected from normalized artifacts. It disables
configuration, cookies, plugins, downloads, external commands, JavaScript runtimes, and remote
components; restricts transport to HTTPS identity responses from reviewed host families with
public-address DNS and redirect validation; and enforces network, redirect, output, file, CPU,
and time bounds, plus a best-effort Linux address-space ceiling. Podcast enclosure URLs are observed
but never fetched automatically.
Provider/worker return types, source identity, media types, finite numeric values, field counts,
output sizes, and consumed budgets are validated at runtime; unexpected provider exceptions become
generic diagnostics rather than exposing provider exception text.

The media extra supplies yt-dlp to the built-in worker. Because it is an optional dependency,
`youtube_metadata` remains `optional`; absence produces `DEPENDENCY_MISSING`, and lock-file presence
is not evidence that a live target succeeded.

## Snapshot and archive boundaries

`SnapshotStore` keeps immutable sanitized JSON metadata beside bodies in the filesystem CAS.
`CacheKey` partitions by URL, representation, authentication scope, policy profile, language,
region, parser version, and relevant `Vary` values. The opaque authentication reference itself is
never persisted.

Lookup has four explicit states: `MISS`, `FRESH`, `STALE_WHILE_REVALIDATE`, and `REVALIDATE`.
Conditional `ETag` and `Last-Modified` validators can create a new immutable record for a verified
not-modified response. Every read rechecks artifact hash, size, resource lineage, cache partition,
and metadata bounds. Public and authenticated records cannot collide.

Internet Archive acquisition uses the built-in `WaybackSnapshotConnector`. It performs a bounded
availability lookup on exact `archive.org`, upgrades only the returned exact-host capture locator,
then retrieves an exact `web.archive.org` raw capture while reapplying destination and redirect
policy with pinned addresses at every hop. The capture metadata must bind the requested original
URL and timestamp exactly. Other remote search-cache, archive, and CDN paths use injected
`SnapshotConnector` implementations. Every connector receives a public sanitized target plus the
remaining byte/deadline limits. Python validates returned type, original-source authority, HTTPS
snapshot locator, media type, body size, and capability-specific origins. Authenticated/private
requests are rejected before any third-party connector use.

Connector bodies pass the normal text/binary quality assessment. Low-quality responses are retained
as checked-only artifacts, recorded as unsuccessful snapshots, and cannot short-circuit a later
fallback. Native cache stores likewise admit only accepted artifacts with the exact required
representation. Cache-only plans produce `clean_text` before RAG storage, `rendered_html` before
browser storage, and configured `search_results` before either search cache; raw HTTP cannot
silently populate those partitions.

`ArchiveAdapter` parses ZIP/TAR through `python -m fetech.archive_worker`. The worker denies path
traversal, absolute paths, NULs, duplicate names, links/devices, nested archives, excessive members,
expanded bytes, and compression ratios. The parent validates the complete returned member manifest
before writing individual immutable artifacts.

On POSIX systems the common runner uses a separate bootstrap process and process group. Startup has
an independent deadline, applies irreversible CPU/output/file-size/file-descriptor/core-dump bounds,
then replaces itself with the fixed worker command; worker communication receives only the
remaining total wall budget. In explicit `development` mode this remains resource control rather
than an operating-system sandbox.

Linux `required` mode adds one delegated cgroup-v2 leaf per covered built-in invocation, explicit Bubblewrap
user/mount/PID/IPC/UTS/cgroup/network namespaces, minimal read-only mounts, size-limited private
tmpfs, `no_new_privileges`, and an inner libseccomp/rlimit bootstrap. The parent waits for both the
Bubblewrap child event and the inner containment marker before sending input, and `cgroup.kill`
removes descendants on every exit path. Document, archive, image, browser, FFmpeg, FFprobe, and
Tesseract profiles deny network access at the kernel namespace. Native media tools do not receive
the Python audit hook, so the required Linux boundary is their authoritative containment.
Logic engines, optional curl HTTP/3, local yt-dlp, and injected providers are outside these
profiles and must be isolated separately, brokered where applicable, or disabled.

## Failure and provenance semantics

- Missing optional engines/providers: `DEPENDENCY_MISSING`.
- Missing or invalid required Linux containment: typed dependency/execution failure before worker
  input, never an unsandboxed retry.
- Local yt-dlp in required mode: unavailable until an allowlisting egress broker exists.
- Unsafe or malformed input: `FAILED` with a sanitized capability-specific failure code.
- Private use of a third-party snapshot connector: `BLOCKED_BY_POLICY`.
- Textless scanned PDF without a configured OCR provider: checked-only `NEEDS_OCR`; validated
  provider output may satisfy the request.
- Cache miss or failed integrity verification: typed failure, never accepted evidence.
- Low-quality remote snapshot: checked-only result and unsuccessful cache record, never accepted
  evidence.
- Every derived artifact keeps its raw source parent and original publisher authority.
- Worker/provider diagnostics omit bodies, credentials, signed URLs, and sensitive headers.
