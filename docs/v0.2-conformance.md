# v0.2 conformance

Fetech v0.2 closes 40 registered implementation paths: 12 discovery and crawl capabilities, 13 URL
alternatives, and 15 browser capabilities. Together with v0.1, the runtime exposes 96 cumulative
paths. `GET /v1/capabilities` and `fetech capabilities` return the authoritative status overlay.

## Discovery and crawling

`intent=crawl` plans one root HTTP acquisition followed by `depth_limited_crawl`. The discovery
adapter reuses the policy-checked HTTP boundary for every frontier item, executes bounded batches,
deduplicates normalized URLs, and admits only the root site. It prioritizes `/sitemap.xml`, extracts
HTML navigation relationships and sitemap `<loc>` values, and optionally queries a configured search
connector. Search results do not bypass same-domain or depth checks.

`ResourceBudget.crawl_pages` counts checked pages, including failures. Attempts, wire bytes,
decompressed bytes, redirects, deadline, per-host pacing, and global/per-host concurrency also remain
bounded. The returned `CrawlReport` records fetched and failed targets, parent relationships, maximum
depth, and omitted frontier size. A matching immutable JSON artifact preserves the report lineage.

## URL alternatives

The candidate stage deterministically projects all 13 registered generator outcomes into a sanitized
`url_candidates` artifact. It supports HTTP-to-HTTPS, www/non-www, slash, tracking-query cleanup,
observed canonical, mobile, AMP, print, language, and region candidates. `https_to_http` is always
`BLOCKED` and never materializes a URL.

If visible text from the original HTML is inadequate, the stage may spend remaining attempt and byte
budget on candidates in deterministic order. Each attempt is revalidated by the HTTP policy. Variant
acquisition is disabled for private, authenticated, and sensitive-query requests. The original
request URL remains the authority for selected variant resources.

## Browser rendering

The local Playwright path runs in a bounded subprocess over acquired HTML. JavaScript is enabled,
while offline mode, service-worker blocking, and route abortion prevent page scripts from opening
network connections. The worker can produce rendered DOM, visible text, and a bounded PNG while
applying request-selected selector waits, network-idle waits, scrolling/lazy loading, expansion
clicks, cookie-banner actions, and SPA history observation.

Playwright/Chromium is an optional installation. Puppeteer and Selenium are optional service
connectors implementing the same JSON render contract. Connector use requires an HTTPS operator
endpoint plus `policy_profile=allow_remote_browsers`; private, authenticated, or sensitive targets
are rejected. A missing binary or connector returns `DEPENDENCY_MISSING` without weakening static
fetching.

## Verification surface

The hermetic suite checks cardinality and status, deterministic planning, bounded crawl success and
failure, domain/depth enforcement, optional search, no-downgrade candidates, visible-text fallback,
authority preservation, explicit browser planning, all 15 browser outcomes, screenshots, and HTTPS
offline connector requests. Actual Playwright execution is exercised when the browser extra and a
Chromium binary are installed; otherwise its dependency failure remains typed and tested.
