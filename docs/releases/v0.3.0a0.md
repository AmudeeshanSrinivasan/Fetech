# Fetech v0.3.0a0

Fetech v0.3.0a0 is an alpha prerelease that closes the authentication/private-session and
structured-API/feed categories. It provides 23/23 v0.3 implementation paths and 119/155 cumulative
capability paths across the first nine categories.

## Highlights

- Origin-scoped credential and session providers with opaque references, per-hop credential
  stripping, authenticated-cache isolation, expiry handling, and bounded OAuth/SSO refresh.
- Dual-approved, one-shot form submission with bounded CSRF extraction and request-local login
  cookie handoff.
- Safe normalization for REST, GraphQL, JSON, XML, RSS, Atom, sitemaps, OpenAPI, and five named
  public-API families.
- Python remains the required security and execution authority. Optional Clingo planning and
  SWI-Prolog explanation backends are bounded, schema-validated, and fall back to Python.
- A reproducible SPDX 2.3 SBOM, dependency-license report, security threat model, and evidence-based
  competitor matrix accompany the release.

The configured optional v0.3 paths are `sso` and `private_workspace`. Their typed connector
boundaries, policy checks, outcomes, and tests ship in core, while an operator must supply the
identity or workspace connector.

## Known alpha constraints

- Packaged CLI, daemon, and MCP entry points use null credential/session providers. Authenticated
  execution requires an explicitly embedded provider.
- Authenticated HTTP/3 fails closed because the bounded curl path has no secret-safe credential
  channel.
- SWI-Prolog and curl are explicitly installed system executables. Browser engines also require
  their separately installed runtime binaries.
- The local logic subprocess runner is resource bounded, not a complete operating-system sandbox.
  Production deployments must add worker or container isolation.
- The daemon and MCP server have no built-in multi-user authentication or artifact ACL. Keep them
  on loopback or behind an authenticating and authorizing reverse proxy, and treat the complete data
  directory as potentially confidential.
- Document, OCR, media, archive, and full snapshot closure remains scheduled for v0.4. This is not
  the 155/155 v1.0 release.
